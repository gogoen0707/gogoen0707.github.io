정보보호 시스템의 이해

  • 시스템 보안

    :사용 허가권이 없는 사용자가 파일, 라이브러리 폴더 및 장치 등을 사용하지 못하도록 제한하여 보호하는 시스템 기능.

    • - 시스템 :어떤 과업의 수행이나 목적 달성을 위해 공동 작업하는 조직화된 구성 요소의 집합.

    -
  • 정보보안 3대 요소
  • :
    기밀성(Confidentiality): 인가(허락)된 사용자만​ 정보자산에 접근가능 해야한다.
    무결성(Integrity): 인가(허락)된 사용자만 정보자산에 대해 변경가능 해야한다.
    가용성(Availability): 정보자산에 대해 사용자가 필요한 시점에 사용가능 해야한다.

    - 시스템 보호 장치 :
    접근 제어: 자원이 어떻게 접근되는지 제어 -> 인가되지 않은 수정이나 노출로부터 보호

  • 네트워크 보안


    • - 네트워크 : 시스템 간의 데이터 연결 및 전송의 통신망 -> 시스템 내: 강한 연결 / 시스템 외: 약한 연결

    - 네트워크 주소 체계 : 인터넷 계층 -> IP 주소: 각각의 컴퓨터를 네트워크에서 유일하게 구분해 줄 수 있는 고유 번호

    네트워크 ID -> IP 주소= Network ID + Host ID

    IP 주소 표기: 점으로 구분되는 4개의 십진수 -> 각 바이트; 0과 255 사이의 십진수

    - 네트워크 공격과 보호 방법 :

    네트워크 공격 :
    Dos: 공격자가 서비스 제공에 필요한 자원들을 정상적 사용자에게 허락하지 않게 하기 위해 가짜 서비스 요청을 비정상적으로 하는것
    스니핑: A, B의 통신 내용을 C가 잃는 것이 가능하게 만드는 것
    IP Spoofing: 가짜 주소로 공격자가 메시지를 보내는 것

    네트워크 보호 방법 :
    방화벽(fireplace): 기관의 내부 네트워크를 구분하기 위해 네트워크를 구분하는 것 (내부와 외부를 구분)
    응용 플록시: 메시지의 내용 확인
    IDS: 방화벽을 뚫고 들어온 공격자를 지속적으로 모니터링하기 위한 장치들을 다양한 위치에 설치
